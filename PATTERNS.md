CREATIONAL PATTERN
=================== 
    1. SINGLETON
    Singleton pattern is like a object declare in the globally and can be accessed on need. For eg. database connection server
    2. FACTORY
    Factory pattern maintain all methods inside its function. It will be systematic and complex. It will add a abstract layer. It turns the messy code into a readable format.
    3. BUILDER
    Builder patter is like adding a methods one by one based on the need instead of throwing everything in the plate in a single shot.
STRUCTURAL PATTERN - Object interact with each other
==================
    1. FASCADE
    Fascade pattern is like it holds the entire object and it wont reveal outside. The works would be done internally.For example. in ecommerce application the cart and buy functionality dont reveal the transaction complexity to the end user.
    2. ADAPTER
    To achieve the requirement there is a dependency of thrid party pieces that helps to achieve the expectation.

BEHAVIOUR PATTERN
===================
1. STRATEGY 
Strategy pattern is making the code change at interface level without affecting the goal.It works perfect for open closed principle.
